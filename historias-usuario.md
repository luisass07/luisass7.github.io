# Historias de Usuario – ERP PolloSoft

## Historia de Usuario – Módulo de Compras
Como administrador  
Quiero registrar órdenes de compra a proveedores  
Para llevar control de los productos adquiridos por la empresa.

### Criterios de aceptación
- El sistema debe permitir seleccionar el proveedor.
- Se deben agregar productos y cantidades.
- El sistema debe calcular el total de la compra.
- La orden debe guardarse en el sistema.


## Historia de Usuario – Módulo de Facturación
Como cajero o administrador  
Quiero generar facturas de venta  
Para registrar las ventas realizadas a los clientes.

### Criterios de aceptación
- El sistema debe permitir seleccionar productos.
- Debe calcular subtotal, impuestos y total.
- La factura debe quedar registrada.
- Debe poder imprimirse o descargarse.


## Historia de Usuario – Módulo de Stock / Costos
Como encargado de inventario  
Quiero visualizar el stock disponible de los productos  
Para controlar la disponibilidad y costos.

### Criterios de aceptación
- El sistema debe mostrar la cantidad disponible.
- Debe mostrar el costo del producto.
- Debe actualizarse cuando haya compras o ventas.
- Debe permitir buscar productos.



## Historia de Usuario 4 – Módulo de Activos Fijos
Como administrador  
Quiero registrar los activos de la empresa  
Para llevar control de los bienes.

### Criterios de aceptación
- El sistema debe permitir registrar nombre del activo.
- Debe guardar fecha de compra y valor.
- Debe permitir consultar los activos registrados.
- La información debe quedar almacenada.



## Historia de Usuario – Módulo de Empleados
Como administrador de recursos humanos  
Quiero registrar empleados  
Para gestionar la información del personal.

### Criterios de aceptación
- El sistema debe permitir registrar nombre, documento y cargo.
- Debe guardar la información en el sistema.
- Debe permitir consultar empleados.
- Debe permitir actualizar información.
